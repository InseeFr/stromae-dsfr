L'affichage

La structure de l'affichage​

L'affichage au sein de l'application peut être découpé de la manière suivante : une entête, du contenu et un pied de page.

Entête​

L'entête est celle imposée par le design système de l'État.

Il est possible de personnaliser le logo et le texte principal ("Filière d'enquête") via les métadonnées.

À l'avenir, nous souhaitons également personnaliser le texte "Application de collecte internet".

  • Le bouton Contacter l'assistance redirige vers : {url_portail}/{pathAssistance}, l'URL du portail étant une variable d'environnement et le chemin Assistance un paramètre de requête.
  • Le bouton Se déconnecter déconnecte l'utilisateur, en demandant au serveur d'authentification de rediriger l'utilisateur vers : {url_portail}/{pathLogout} avec pathLogout également un paramètre de requête.

Par exemple, si au moment du clic sur les boutons mon URL est :

example.url/?pathLogout=log&pathAssistance=assistance/log

La redirection vers l'assistance sera faite vers {url_portail}/assistance/log et la déconnexion redirigera vers {url_portail}/log.

Pied de page​

Le pied de page est celui imposé par le système de design de l'État.

Il est possible de personnaliser le logo de l'Insee et d'ajouter des logos de partenaires.

Contenu​

Le contenu varie en fonction de la page consultée. Il s'agit essentiellement de l'affichage du questionnaire ou d'un message d'erreur.

Erreurs​

En cas de problème, des messages d'erreur sont affichés pour guider l'utilisateur.

Quand l'erreur se produit dans un contexte de collecte, la redirection proposée est vers le portail. Pour la visualisation, c'est vers le formulaire de visualisation, et vers la page d'accueil dans les autres cas.
